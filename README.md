# 台風トレース検索 (Typhoon Track Similarity Console)

地図上でエリア（と、任意で自分の経路）を指定すると、気象庁のベストトラックデータの中から**条件に合う過去の台風**をランキング形式で検索できるWebツールです。サーバー不要・単一HTMLファイルで動作します。

![status](https://img.shields.io/badge/type-static%20HTML-4fd8cf) ![data](https://img.shields.io/badge/data-JMA%20Best%20Track-e8a33d)

**🔗 台風トレース検索：https://awg-yk-cloud.github.io/typhoon_track_finder/**

## できること

- **エリアで絞り込む（必須）**：地図上をドラッグして矩形を指定し、その範囲を通過した台風だけを検索対象にする
- **経路を自由に描く（任意）**：地図をクリックして好きな経路（折れ線）を作成し、ダブルクリックで確定。経路を描かずエリア指定だけでも検索可能（該当台風が12件以下の場合）
- **向き（進行方向）の考慮切り替え**：経路の向き（進行方向）を無視して形だけで比較するか選択可能（既定：無視する）
- **3種類の類似度指標から選択**
  | 指標 | 特徴 |
  |---|---|
  | チャムファー距離 | 双方向の最近傍点距離の平均。外れ値に強く安定した「形」の近さを評価（既定値） |
  | 平均乖離幅（弧長正規化） | 経路全体の平均的なズレの幅を重視 |
  | RMS乖離幅（弧長正規化） | 平均とチャムファー距離の中間的な性質。局所的な大きなズレに敏感 |
- **詳細フィルタ**：中心気圧（この値以下）・勢力（最大風速：強い以上／非常に強い以上／猛烈）・通過月（複数選択可）で候補を絞り込み
- **結果の並び替え**：検索順（デフォルト）のほか、中心気圧・勢力・年で並び替え
- **候補経路のハイライト表示**：結果一覧から候補台風を選択すると、地図上で実際に比較に使った区間（エリア内に厳密に収まる区間）を強調表示
- 各台風の詳細（ウェザーニュース）へのリンク付き

## 必要なもの

このHTMLファイル自体には台風の座標データは含まれていません。以下のデータファイルを**同じフォルダに配置**する必要があります。

- **ファイル名**：`bst_all.txt`
- **入手元**：気象庁 RSMC Tokyo－Typhoon Center「ベストトラックデータ（テキスト形式）」
  https://www.jma.go.jp/jma/jma-eng/jma-center/rsmc-hp-pub-eg/besttrack.html
- ダウンロードした台風ごとのテキストを1つのファイルに結合し、`bst_all.txt` という名前で本HTMLと同じディレクトリに置いてください。
- 対象期間は1951年〜2026年（西暦2桁表記の変換ルールに基づく）です。

> **注意**：`file://` で直接開くとブラウザの制限により `bst_all.txt` を読み込めません。必ずローカルサーバー経由（例：`python -m http.server` やGitHub Pages）で開いてください。

## 使い方

1. `index.html`（本ファイル）と `bst_all.txt` を同じフォルダに置き、Webサーバー経由で開く
2. 「エリアを描く」を押し、地図上をドラッグして探したい海域（矩形）を指定する（検索には必須）
3. 必要なら「経路を描く」を押し、地図上をクリックして経路を作成 → ダブルクリックで確定（経路を描かずエリアのみで検索することも可能）
4. 「詳細設定」で中心気圧・勢力・通過月・類似度指標を調整（任意）
5. 「類似経路を検索」を押すと、右側に条件に合う過去の台風がランキング表示される
6. 結果をクリックすると、その台風の経路（エリア内に収まる区間）が地図上にハイライト表示される

## 技術構成

- 地図表示：[Leaflet.js](https://leafletjs.com/)
- ベースマップ：国土地理院タイル（淡色地図）
- フレームワーク不使用、Vanilla JS + 単一HTMLファイル
- 台風データのパース・各種距離計算（チャムファー距離／平均乖離幅／RMS乖離幅等）はすべてブラウザ内で完結
